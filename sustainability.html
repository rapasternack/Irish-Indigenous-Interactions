Sustainability & Digital Preservation

This page documents how the project is structured for long-term preservation, accessibility, and reuse. It treats sustainability as ensuring that digital cultural records remain readable, portable, and recoverable over time.

Project Attribution

Created by Rachel Pasternack as part of Binghamton University’s Digital History course (Spring 2026). This project examines Irish–Indigenous interactions through histories of solidarity, colonial entanglement, and cross-cultural memory.

The site synthesizes primary archival materials, Indigenous nation records, and peer-reviewed scholarship. All materials are used for educational and non-commercial research purposes.

Primary Archival Sources

  • Library of Congress — treaties, newspapers, and migration records
  • British Library — Irish and colonial-era documentation
  • UK National Archives
  • National Library of Ireland

Indigenous Nation Sources

  • Choctaw Nation — famine relief and solidarity archives
  • Navajo Nation — COVID-19 mutual aid records
  • Native Land Digital — territorial visualization database

Irish Institutional Sources

  • Oireachtas
  • National Museum of Ireland
  • Dublin City Council
  • Irish Famine Archive

Secondary Scholarship

  • Posey, D. A. — Indigenous Peoples and Sustainability
  • McGreavy et al. (2021) — Indigenous environmental governance research
  • MacNeill — frameworks of sustainable development in Indigenous contexts
  • Digital Public Library of America (DPLA) collections

Technical Platform

This project is hosted on GitHub Pages, a static-site deployment system that publishes directly from a repository. This infrastructure was selected for its long-term stability, transparency, and ease of replication.

The site is intentionally built as a static HTML/CSS system. This design choice minimizes technical dependencies, ensuring that the project can remain functional even as web frameworks evolve or become obsolete.

Essential Files

The project is designed to allow future users to reconstruct it without loss of meaning or structure.

  • HTML — structural and narrative content for each webpage
  • CSS — visual and typographic system for style sheet
  • JavaScript / JSON — timeline and database logic
  • Assets folder — images and media
  • Documentation — contextual and methodological notes

Future Maintenance Requirements

A future maintainer should be able to update the project without compromising structure or interpretive integrity.

  • Basic understanding of HTML/CSS structure
  • Access to GitHub repository
  • Ability to maintain external links and media integrity
  • Optional GIS platform knowledge for map updates
  • Awareness of page hierarchy and navigation system

Ongoing Maintenance Strategy

  • Regular verification of external links and archival sources
  • Updates to reflect new scholarship or corrected historical interpretations
  • Expansion of timeline, database, and map components
  • Preservation of design consistency across all pages
