Platform and Structure

This project was built using GitHub pages, a free hosting platform that published websites directly from a GitHub repository (which can be found at the footer). This platform was chosen because it ensures long-term accessibility and easy collaboration for future users. GitHub also allows the project to be cloned or downloaded, which supports preservation and reuse. This site uses a simple static structure (HTML and CSS), which further improves sustainability because it does not rely on complex frameworks. Static sites are easier to maintain over time and are less likely to break as technology changes.

Essential Project Files

To fully preserve this project, the follow files and folders are needed:

  • HTML files – Serve as the core content and structure
  • index.html – Homepage introducing the project
  • findings.html – Timeline, map, and archival materials
  • planning.html – Research design and methodology
  • sustainability.html – Technical documentation and maintenance
  • CSS files – Layout and visual design
  • style.css – Global styling rules
  • Image Folder – Visual content
  • assets/ – Images and media files
  • Data Files – .csv that contains timeline and map content
  • Repository Documentation – Contains contextual information about the project

What a Future User Would Need

A future maintainer should be able to update or migrate this project without breaking structure or content.

  • Basic knowledge of HTML and CSS
  • Access to the GitHub repository
  • ArcGIS account (only for map updates)
  • Local copies of images or verified external URLs
  • Understanding of navigation structure across pages
  • External resources such as CSS frameworks and fonts

Maintenance Notes

  • Periodic checks for broken links, missing media, outdated content which are then updated or replaced
  • Continued clear documentation to support future maintenance and users
  • Expansion can include adding new entries to the timeline, map, and database and enhancing interactivity
