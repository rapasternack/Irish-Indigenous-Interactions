## Technical Sustainability

### Hosting

**Current hosting**: GitHub Pages

**Long-term plan**:  
After the semester ends, the project will remain publicly accessible via GitHub Pages as long as the repository is maintained. Over the next five years, the site will ideally remain live in its current form as a static digital humanities project. If any active maintenance stops, the stie will still function as an archived resource due to the use of HTML and CSS

**Costs**:  
The are no hosting fees for GitHub Pages and therefore the project is free to host, with no anticipated financial needs.

**Migration plan**:  
If GitHub Pages were discontinued, the project would be migrated to another static hosting platform or a server because it was built with HTML and CSS and is therefore very easily portable.

### Dependencies and Versions

**Current dependencies**:
```
Python 3.12
pandas 2.x
matplotlib 3.x
HTML5
CSS3
```

**Version management**:  
Versions will be documented informally through project notes and repository history and if expanded, then a seperate requirements.txt will be include in order to ensure reproducibility for future users.

**Dependency risks**:  
Due to the main site being static, there is minimal dependency risk. If any external libraries or scripts become deprecated, they can be replaced with updated or even local versions.

### Code and Data Preservation

**Code repository**: GitHub - https://github.com/rapasternack/Digital-History

**Backup locations**:  
Local copies on personal computer. There is potiental future backup in institutional repositories like Binghamton.

**Data archiving**:  
All project data (images, content, text) is planned to be stored within the repository in widel supported formats (JPG, HTML, CSV) to ensure long-term accessibility

**Documentation**:  
The project is planned to include basic documentation within the site and repository with additionally README files helping future user understand.
